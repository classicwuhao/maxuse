Detailed Description

A class invariant is a boolean expression that must hold in every system state for each object of a class.

Expression org.tzi.use.uml.mm.MClassInvariant.expandedExpression ( )
inline

Returns the expanded expression of the invariant. This expression requires no context and can be evaluated globally. If isExistential() is false, it is enclosed by a forAll expression iterating over all instances of a class. Otherwise, it is enclosed by an exists expression.

Expression org.tzi.use.uml.mm.MClassInvariant.getExpressionForSatisfyingInstances ( )
inline

Returns an expression for selecting all instances that satisfy the invariant. The expression is generated as C.allInstances->seject(v1 | <inv>) if it has one iteration variable. For two and more variables, an additional C.allInstances()->forAll(v2| <inv>) is introduced, because select only allows one iteration variable.

Definition at line 241 of file MClassInvariant.java.

Expression org.tzi.use.uml.mm.MClassInvariant.getExpressionForViolatingInstances ( )
inline

Returns an expression for selecting all instances that violate the invariant. The expression is generated as C.allInstances->reject(v1 | <inv>) if it has one iteration variable. For two and more variables, an additional C.allInstances()->forAll(v2| <inv>) is introduced, because reject only allows one iteration variable.
