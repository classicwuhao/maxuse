Detailed Description

A Report formats data as a table. The complete report is built up in memory before it can be printed. This is necessary to determine column widths. Each column may be left-aligned, centered, or right-aligned.

org.tzi.use.util.Report.Report ( int  columns,
String  fmt 
)
inline

Creates a new report with the specified number of columns. The format string defines the layout of each row when the report is printed. The position of a column in the format string is specified by a $ sign followed by either a l character for a left-aligned column, a c character for a centered column, or a r character for a right-aligned column. All other characters are copied verbatim into the output. If there are more format specifiers than columns, the excess specifiers are ignored. If there are fewer format specifiers than columns, all columns with no format specifier are left-aligned.

Example: 

Report r = new Report(4, "[ $l = $r, $r, $c ]");

A possible result is: 

[  foo   =      3,          1.2, false ]
[ foobar = 453453, -1.245345345, true  ]
[  line  =    555,        999.0, true  ]
Exceptions
IllegalArgumentExceptionthe fmt string is not well-formed.
